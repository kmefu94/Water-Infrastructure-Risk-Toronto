## 🔹 2. Ward Data Transformations

### 2.1 Table Extraction (Excel Report Format)

#### Issue

Ward profile workbooks contain multiple embedded tables per sheet (e.g., Population, Dwellings), not a single structured dataset.

#### Planned Approach

1. **Table Detection**

   * Identify section headers (e.g., "Population", "Dwellings")
   * Detect table start rows (Toronto, Ward 1, etc.)
   * Extract until next section or blank separator

2. **Separate DataFrames**

   * Extract each table into its own dataset:

     * `population_age`
     * `dwellings_structural_type`
     * others as needed

3. **Standardization**

   * Rename first column:

     * `age_group`, `dwelling_type`, etc.
   * Clean ward columns
   * Remove empty/repeated rows

4. **Reshaping**

   * Convert wide → long format
   * Target grain:

     * **1 row = 1 ward x category**

5. **Metadata Enrichment**

   * Add:

     * `domain` (population, dwellings, etc.)
     * `measure_type`
     * `year`

6. **Storage**

   * Store outputs in:

     * `data/interim/ward/`
   * Use dictionary structure or individual CSVs
